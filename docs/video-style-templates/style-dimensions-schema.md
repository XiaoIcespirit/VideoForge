# 视频风格模板维度定义规范

这个文件是 `docs/video-style-templates/` 中所有视频风格模板的基础 schema。每个模板可以增加更多维度，但不能少于这里列出的维度。

## 模板头部必填信息

- `# <Style Name>`：人类可读的风格名称。
- `slug`：小写短横线命名。
- `mode`：第一版默认 `design-from-brief`。
- `status`：`draft`、`validated` 或 `deprecated`。
- `适合平台`：例如抖音、B 站、YouTube Shorts、横屏课程视频、通用 Remotion composition。
- `默认规格`：画幅、分辨率方向、预计时长、fps 假设。

## 必填维度

### 风格定位

一句话说明这个模板适合什么视频、什么平台、什么观众，以及要留下什么专业印象。

### 适用场景

说明适合的内容类型，例如 AI 教程、产品发布、代码演示、案例拆解、更新日志、短视频爆点、UI walkthrough。

### 叙事结构

说明开场 hook、问题铺垫、概念解释、案例演示、结果 reveal、总结 CTA 的组织方式。

### 镜头节奏

说明平均镜头长度、快慢变化、信息密度、何处停顿、何处快速切换。

### 画面构图

说明主体布局、文本/画面比例、留白、焦点区域、安全区、横屏/竖屏适配倾向。

### 字幕系统

说明口播字幕、关键词大字、逐词高亮、标签、注释、警告/成功状态文案的规则。

### 转场语言

说明常用转场类型、触发时机、节奏强度，以及不能滥用的效果。

### UI/代码/提示词演示方式

说明浏览器、终端、代码块、prompt 卡片、参数面板、时间线、before/after 对照的使用方式。

### 动效语气

说明技术感、发布会感、实验感、短视频冲击感、开发者工具感、克制感或其他明确组合。

### 色彩与字体方向

说明主色方向、强调色策略、背景策略、对比度、字体气质和可读性要求。除非 brief 要求，不硬编码完整品牌色。

### 声音与口播假设

说明是否依赖口播、静音是否可理解、音乐如何配合节奏、音效点落在哪里。

### Remotion 组件映射

说明推荐组件类型和组件来源方向。必须区分 `已验证具体组件` 与 `建议组件类型`，不能虚构组件名。

### 场景结构建议

提供可直接变成 Remotion 分镜的 scene list，包含场景目的、预计时长、画面动作、文字层和动效语言。

### 复用边界

说明哪些部分可以跨视频复用，哪些部分必须随主题、平台或单集内容变化。

### 质量检查清单

列出实现后的检查项：字幕可读性、节奏、信息密度、转场一致性、组件适配、移动端裁切风险、静音观看可理解性。
