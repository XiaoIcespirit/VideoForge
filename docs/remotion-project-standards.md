# Remotion 项目规范

这个文件是本项目后续修改 Remotion 模板、episode 数据、角色素材和渲染输出时必须优先遵守的项目规范。目标是把它稳定做成“IP 角色讲课 + 信息面板 + 动态关键词 + 可批量复用”的短视频模板框架。

## 0. 改动前先看这里

改动以下内容前，先阅读本文件：

- `src/templates/` 下的模板组件、布局、动画和字幕组件。
- `src/episodes/` 下的每集文案、镜头、流程图、关键词和字幕数据。
- `public/ip/` 下的角色图、三视图素材和角色替换方案。
- Remotion still / render 的关键帧检查、封面图和成片导出。

## 1. 背景必须干净

背景层必须保持干净、轻量、有层次，但不能出现无意义的环境文字标签。

硬性禁止：

- 默认背景里不得出现 `SHIP`、`VERIFY`、`AI RUN`、`INTENT` 这类装饰标签。
- 不得把调试标签、占位标签、英文流程词直接散落在背景上。
- 不得为了“科技感”添加与本集知识点无关的漂浮文字。
- 不得让背景装饰干扰主标题、信息卡、角色图卡或字幕区。

允许：

- 干净的浅色渐变、柔和色块、细网格、细线条、弱透明几何形。
- 与当前镜头内容直接相关的流程词，但必须放在信息卡、流程图、关键词模块等明确内容区域里。
- 用户明确要求的背景文案，但必须能解释其信息目的。

实现约束：

- `brand.ambientChips` 默认必须是空数组。
- 如果后续真的需要环境标签，必须在 episode 数据里显式配置，并确认它们是当前内容的一部分。
- 新增背景装饰时，优先使用无文字图形元素。

## 2. 角色贴图规范

角色要有存在感，但默认要以透明贴图的方式服务信息表达，不要给人物外面套可见方框。

- 角色优先放在左侧、左下或画面下半区，不要遮挡主标题、流程图、分栏卡片和字幕。
- 三视图素材要通过 `character.image`、`sourceAspect`、`viewsPerSheet` 参数配置，不能写死在模板组件里。
- 角色周围默认不能出现白底、边框、卡片底、内阴影、外部投影等“方框感”或“贴纸阴影”效果；只有用户明确要求角色卡片时才允许加可见容器。
- 角色贴图必须避免头部、脚部、裙摆、尾巴等关键部位被裁掉。
- 角色需要有轻微呼吸、浮动、缩放或摆动，不能整段完全静止。
- 复杂骨骼动画不是优先事项，先保证裁切干净、节奏清楚、陪讲感成立。

## 3. 模板必须可批量复用

模板组件负责结构和动效，episode 数据负责内容。

可替换项必须继续保持数据化：

- `meta.compositionId`
- `meta.durationSeconds`
- `brand.chapterTitle`
- `brand.episodeLabel`
- `brand.subtitleLabel`
- `character.image`
- `character.sourceAspect`
- `character.viewsPerSheet`
- `highlightWords`
- `scenes`
- `captions`

不要在模板组件里硬编码某一集专属文案。某一集专属内容应该放进 `src/episodes/`。

## 4. 竖屏画面规范

- 画幅保持 1080x1920，9:16。
- 主信息区、角色贴图、字幕区必须有稳定安全边距。
- 字幕区优先放底部，不能被角色或信息卡遮挡。
- 移动端阅读优先：标题短、字大、断句自然。
- 画面不能做成传统黑板讲课，也不能做成纯静态 PPT。
- 信息卡、流程图、关键词必须有清晰层级，避免一屏塞太满。

## 5. 动画与节奏规范

- 前 3 秒必须有强钩子。
- 全片每 2 到 4 秒至少有一次明显变化。
- 优先使用简单有效的动效：平移、缩放、渐入、弹出、流程高亮、卡片滑入。
- 镜头可以轻推，但不要剧烈晃动。
- 动画要服务理解，不为炫技牺牲可读性。

## 6. 字幕规范

- 全程必须有字幕。
- 字幕断句要贴近口语，不要一行堆太长。
- 重点词可以高亮，但高亮词必须来自 `highlightWords` 或场景数据。
- 字幕不能与底部安全区、角色贴图和信息卡发生视觉冲突。

## 7. 素材与输出路径规范

- IP 素材放在 `public/ip/`。
- 如果 IP 原图来自黑底或深色底，不能只做硬阈值抠图；先使用 `.agents/skills/clean-character-cutout` 生成带羽化和去黑边处理的透明版，再由模板引用。
- 项目内可复用生成素材放在 `public/generated/`，按 `images/`、`videos/`、`audio/`、`captions/` 分类。
- Remotion 关键帧、封面图和成片输出放在 `out/` 子目录，不要直接堆在 `out/` 根目录。
- 成片和草稿视频放在 `out/renders/epNN/`。
- still、关键帧检查图放在 `out/stills/epNN/`。
- 封面和对外预览图放在 `out/previews/epNN/`。
- 抠图、模型对比、清理实验和 contact sheet 放在 `out/diagnostics/<task>/`。
- `video-understanding` 原始分析结果默认放在 `docs/video-analysis/raw/`，从分析中提炼出的可复用风格模板放在 `docs/video-style-templates/`。
- 更完整的路径规范见 `docs/output-structure.md`。
- 不要把只存在于临时目录的素材写进 episode 或模板引用。
- 不要覆盖用户给的原始 IP 图，处理版要另存为清晰的新文件名。

## 8. 修改后的检查清单

涉及视觉、布局、角色或背景的改动完成后，至少检查：

- 背景是否干净，确认没有 `SHIP / VERIFY / AI RUN / INTENT` 这类环境标签。
- 角色贴图是否完整，没有明显裁头、裁脚、可见方框或遮挡字幕。
- 关键镜头中文字是否清楚，字幕是否能读。
- 画面是否仍然像“IP 讲课短视频”，不是黑板课，也不是静态 PPT。
- 如改动较大，渲染 2 到 4 张关键帧 still 预览。
- 最后运行 `npm run lint`，除非本次只改文档。
